# System Patterns — 10and10

## Architecture Patterns

### Next.js App Router conventions
- Server Components by default — `'use client'` only when needed (events, state, DOM)
- `generateMetadata` on all watch and brand pages (SEO requirement from day 1)
- `generateStaticParams` for pre-rendering seed content
- ISR with `revalidate: 3600` for pages with active reviews (Phase 2+)

### Slug generation (D1 — decided)
Pattern: `[brand-slug]-[commercial-name-slug]-[reference-slug]`
Example: `rolex-submariner-116610ln`
- If no commercial name: `rolex-116610ln`
- Generated by utility function `generateWatchSlug(brand, commercialName, reference)`
- Stored in DB, not computed at runtime
- Single source of truth — never recompute from parts

### Prisma patterns
- UUID primary keys: `@id @default(uuid())`
- Timestamps on every model: `createdAt DateTime @default(now())` + `updatedAt DateTime @updatedAt`
- Denormalized counters: `averageRating Float?` and `reviewCount Int @default(0)` on Watch
- Updated on every Review write (not computed at query time)

### API patterns
- `/api/[resource]` — REST-style routes
- Auth via Clerk middleware — `currentUser()` in Server Components, `auth()` in API routes
- Meilisearch proxy at `/api/search` — never expose Meilisearch directly to client

### Meilisearch index config
- `watches` index: searchable `[name, commercialName, reference, description, brandName]`
- Filterable: `[brandId, caseMaterial, dialColor, yearIntroduced, movementType]`
- Sortable: `[yearIntroduced, averageRating]`

## UI/UX Patterns

### Watch detail page layout
- Desktop: 60/40 split (gallery / specs + CTAs), tabs below
- Mobile: tabs (image+basics / specs / movement / reviews)
- Tab pattern used to handle data density on small screens

### Component conventions
- `SpecsTable`: two-column, label `stone-500 uppercase tracking-wide text-xs` / value `stone-900`
- `WatchCard`: always square image (aspect 1:1), overlay on hover
- All images via `next/image` + Cloudinary transforms

## Data Patterns

### Watch data quality rule
Every watch must have: `reference`, `brandId`, `slug`. Everything else is optional but encouraged.
Incomplete watches are visible but marked with a visual indicator.

### Cloudinary folder structure
`10and10/watches/[watch-slug]/main.jpg` (primary)
`10and10/watches/[watch-slug]/gallery-NN.jpg` (gallery)
`10and10/brands/[brand-slug]/logo.svg`
